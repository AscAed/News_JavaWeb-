# /run-tests-and-fix

自动运行项目测试，分析失败原因，并尝试修复，然后获得用户审查。

1. **检测测试命令**：根据项目配置自动识别测试命令（npm test / pytest / mvn test 等）
2. **运行测试**：执行完整测试套件并捕获输出
3. **分析失败**：
   - 对每个失败的测试，检查测试代码和相关实现代码
   - 识别根本原因（逻辑错误、类型不匹配等）
   - 提出最小化修复方案
4. **应用修复**：修改相关文件，只修改必要部分
5. **重新测试**：运行失败的测试子集，确认问题解决
6. **生成报告**：
   - 运行了哪些测试
   - 修复了哪些失败及修复方式
   - 剩余未解决的问题和建议

**调用命令**：`/run-tests-and-fix`
