# Security Rules

## 输入验证
- 验证所有外部输入（HTTP请求、CLI参数、环境变量）
- 在信任边界处进行数据清理和过滤
- 使用 ORM 或查询构建器，避免原生 SQL

## 认证与授权
- 所有受保护端点必须验证身份
- 使用既有的中间件或授权助手，不要自己实现
- 密钥、令牌、密码永远不要提交到代码库
- 使用环境变量或专用的密钥管理方案

## 日志与监控
- 记录安全相关事件（登录失败、权限拒绝），不要记录敏感数据
- 避免在错误消息中泄露系统信息

## 依赖管理
- 定期运行 `npm audit`、`pip-audit` 检查安全漏洞
- 只在明确要求时才添加新依赖
